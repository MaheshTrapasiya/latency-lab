# Security Policy

## Supported versions

Security fixes are provided for the latest published version of latency-lab.
Users should upgrade to the latest npm release before reporting an issue.

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting:

https://github.com/MaheshTrapasiya/latency-lab/security/advisories/new

Include the affected version, impact, reproduction steps, and any suggested
mitigation. Do not disclose the issue publicly until a fix has been released.

You should receive an initial response within seven days. Confirmed issues will
be addressed according to severity and coordinated with the reporter before
public disclosure.

## Scope

Reports should concern latency-lab source code or published npm artifacts.
Vulnerabilities in development-only dependencies that cannot affect consumers
may be handled as maintenance updates rather than package advisories.
